# 架构设计

## 🏗️ 系统架构

LobsterPress 采用**模块化微服务架构**，由五个核心模块组成：

```
┌─────────────────────────────────────────────────────────┐
│                    LobsterPress                         │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │         1. 消息重要性评估引擎                    │   │
│  │         Message Importance Engine                │   │
│  └─────────────────────────────────────────────────┘   │
│                       ↓                                │
│  ┌─────────────────────────────────────────────────┐   │
│  │         2. 核心压缩引擎                          │   │
│  │         Context Compressor v5                   │   │
│  └─────────────────────────────────────────────────┘   │
│                       ↓                                │
│  ┌─────────────────────────────────────────────────┐   │
│  │         3. 自适应学习引擎                        │   │
│  │         Adaptive Learning Engine                 │   │
│  └─────────────────────────────────────────────────┘   │
│                       ↓                                │
│  ┌─────────────────────────────────────────────────┐   │
│  │         4. 预测性压缩引擎                        │   │
│  │         Predictive Compressor                   │   │
│  └─────────────────────────────────────────────────┘   │
│                       ↓                                │
│  ┌─────────────────────────────────────────────────┐   │
│  │         5. 成本优化器                            │   │
│  │         Cost Optimizer                          │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

## 📦 模块说明

### 1. 消息重要性评估引擎

**功能**: 评估每条消息的重要性分数

**算法**: 多维度评分系统
- 关键词检测（决策、错误、配置等）
- 上下文相关性分析
- 时间衰减因子
- 用户行为权重

**输出**: 0-100 的重要性分数

### 2. 核心压缩引擎

**功能**: 根据策略压缩对话历史

**策略**:
- **Light**: 保留 120% 消息（轻微压缩）
- **Medium**: 保留 100% 消息（标准压缩）
- **Heavy**: 保留 80% 消息（深度压缩）

**流程**:
1. 读取会话文件
2. 估算 Token 使用率
3. 评估消息重要性
4. 按重要性排序
5. 保留高分消息
6. 压缩低分消息为摘要
7. 应用压缩结果

### 3. 自适应学习引擎

**功能**: 根据用户行为优化系统参数

**学习维度**:
- 用户关注的消息类型
- 策略使用效果
- 时间模式分析
- 反馈学习

**输出**: 动态调整的权重和策略推荐

### 4. 预测性压缩引擎

**功能**: 提前预测 Token 需求并压缩

**算法**:
- 监控 Token 增长速度
- 预测达到阈值的时间
- 提前执行压缩

**优势**: 避免突发超限，平滑处理

### 5. 成本优化器

**功能**: 分析和优化 API 调用成本

**功能**:
- 统计 API 调用频率
- 分析策略经济性
- 推荐最经济的方案
- 清理缓存

## 🔄 数据流

```
会话输入
   ↓
消息重要性评估
   ↓
Token 使用率检测
   ↓
策略选择（Light/Medium/Heavy）
   ↓
消息排序与筛选
   ↓
生成压缩摘要
   ↓
应用压缩结果
   ↓
记录统计数据
   ↓
更新学习参数
```

## 💾 数据存储

### 压缩历史
```
~/.lobster-press/compression-history.md
```

格式:
```
- 时间戳 | 会话ID | 策略 | 原始大小 → 压缩后大小 | 节省百分比 | 备注
```

### 学习数据
```
~/.lobster-press/adaptive-learning/
├── user-behavior.json       # 用户行为记录
├── compression-stats.json   # 压缩效果统计
├── adaptive-weights.json    # 自适应权重
└── feedback.json            # 用户反馈
```

### 缓存数据
```
/tmp/compress-cache/
├── session_id_1.json
├── session_id_2.json
└── ...
```

## ⏰ 定时任务

| 任务 | 频率 | 功能 |
|------|------|------|
| `lobster-compress` | 每30分钟 | 预测性压缩 |
| `lobster-learning` | 每小时 | 智能学习调度 |
| `lobster-optimizer` | 每天 | 成本优化分析 |

## 🔒 安全设计

### 敏感信息保护
- API Key 不写入代码
- 配置文件权限 600
- 日志不包含敏感数据

### 故障恢复
- 压缩前自动备份
- 失败时自动回滚
- 历史记录追踪

### 限流保护
- API 调用限速
- 失败重试机制
- 错误日志记录

## 📊 性能优化

### 缓存机制
- 评估结果缓存
- Token 估算缓存
- 学习参数缓存

### 批处理优化
- 批量评估消息
- 并发处理会话
- 延迟写入

### 资源控制
- 内存使用 < 10MB
- CPU 占用 < 0.1%
- 磁盘空间 < 5MB
