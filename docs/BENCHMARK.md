## 💰 Token 成本分析

### 压缩开销（每次压缩消耗的 Token）

| 操作 | Token 消耗 | 说明 |
|------|-----------|------|
| 消息重要性评分 | ~500 | 每条消息约 10 Token × 50 条 |
| 生成压缩摘要 | ~800 | 每段摘要约 200 Token × 4 段 |
| 学习权重更新 | ~300 | 自适应学习引擎 |

### ⚠️ 重要发现

压缩开销是**固定成本**（约 1,600 Token），无论上下文多大。

## 💡 使用建议

### 1. 根据上下文长度选择策略

```bash
# < 10k Token：不建议压缩
if [ $CONTEXT_TOKENS -lt 10000 ]; then
  echo "上下文太小，不建议压缩"
fi

# 10k-20k Token：使用 Medium 策略
if [ $CONTEXT_TOKENS -ge 10000 ] && [ $CONTEXT_TOKENS -lt 20000 ]; then
  lobster compress --strategy medium
fi

# > 20k Token：使用 Heavy 策略
if [ $CONTEXT_TOKENS -ge 20000 ]; then
  lobster compress --strategy heavy
fi
```

### 2. 不要频繁压缩

- 建议：上下文超过 60% 时再压缩
- 不建议：每次对话后都压缩（开销累积）

### 3. 结合 OpenClaw Compaction

- OpenClaw 自带 Compaction 功能（免费）
- LobsterPress 提供更精细的控制（但需成本）
- 建议：先用 OpenClaw Compaction，超过 80% 再用 LobsterPress

## 📊 成本计算器

```bash
# 估算你的净收益
context_tokens=30000
strategy="medium"
overhead=1600

case $strategy in
  light) savings_rate=0.12 ;;
  medium) savings_rate=0.27 ;;
  heavy) savings_rate=0.43 ;;
esac

net_savings=$(echo "$context_tokens * $savings_rate - $overhead" | bc)
echo "预计净收益: $net_savings Token"

if [ $net_savings -gt 0 ]; then
  echo "✅ 值得压缩"
else
  echo "❌ 不建议压缩"
fi
```

## 🎯 总结

1. **压缩有固定成本**（约 1,600 Token）
2. **小上下文不值得压缩**（< 10k Token）
3. **大上下文收益显著**（> 20k Token，净节省 6k-18k）
4. **选择合适策略**（Light/Medium/Heavy 根据上下文大小）
