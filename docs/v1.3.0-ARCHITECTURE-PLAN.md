# v1.3.0 架构计划

## 目标

用 Python 完全替代 Shell 的核心逻辑，保留 Shell 仅作为系统集成层。

---

## 当前架构（v1.2.0）

```
├── Shell 脚本（Legacy）
│   ├── context-compressor-v5.sh
│   ├── message-importance-engine.sh
│   ├── adaptive-learning-engine.sh
│   └── cost-optimizer.sh
│
├── Python 脚本（新）
│   ├── lobster_press_v120.py
│   ├── tfidf_scorer.py
│   ├── semantic_dedup.py
│   └── extractive_summarizer.py
│
└── Systemd
    └── 直接调用 Shell 脚本
```

**问题：** 新旧系统并存，边界模糊。

---

## 目标架构（v1.3.0）

```
systemd timer
    └── lobster_runner.sh（轻量 Shell，仅负责调用）
            │
            ├── lobster_press_v130.py（所有核心逻辑）
            │   ├── TF-IDF 评分
            │   ├── 语义去重
            │   ├── 提取式摘要
            │   ├── Token 计数
            │   └── 净收益校验
            │
            └── 输出
                ├── 压缩后的 JSON
                └── 日志
```

## 迁移计划

### Phase 1: 创建统一入口

| 任务 | 文件 | 说明 |
|------|------|------|
| 创建 `lobster_runner.sh` | `scripts/lobster_runner.sh` | 轻量 Shell 调用层 |
| 统一 `lobster_press_v130.py` | `scripts/lobster_press_v130.py` | 整合所有 Python 模块 |

### Phase 2: 更新 Systemd

| 任务 | 文件 | 说明 |
|------|------|------|
| 更新 service 文件 | `systemd/lobster-compress.service` | 调用 `lobster_runner.sh` |
| 添加配置文件 | `/etc/lobster/config.json` | 统一配置管理 |

### Phase 3: 标记 Shell 为 Legacy

| 任务 | 说明 |
|------|------|
| 添加 deprecation 警告 | Shell 脚本启动时提示 |
| 更新 README | 明确 Shell 为 legacy |
| 保留 6 个月 | 给用户迁移时间 |

## API 设计

### `lobster_press_v130.py`

```bash
# 基础用法
python3 lobster_press_v130.py --input messages.json --output compressed.json

# 指定策略
python3 lobster_press_v130.py --input messages.json --strategy medium

# 指定模式
python3 lobster_press_v130.py --input messages.json --mode local

# JSON 输出（供程序调用）
python3 lobster_press_v130.py --input messages.json --format json

# 详细报告
python3 lobster_press_v130.py --input messages.json --report
```

### `lobster_runner.sh`

```bash
# 基础用法（调用 Python）
./lobster_runner.sh --input messages.json

# 指定配置文件
./lobster_runner.sh --config /etc/lobster/config.json

# 调试模式
./lobster_runner.sh --debug
```
